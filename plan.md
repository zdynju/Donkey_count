# 农场驴目标检测与计数项目计划

## 1. 项目目标

设计并实现一个面向农场俯视视角监控图片的驴目标检测算法。系统输入为单张图片或一段时间内的视频/图片序列，输出内容包括：

- 每张画面中驴的位置检测框。
- 每张画面中驴的数量。
- 指定时间段内画面中驴数量的统计结果。
- 可视化结果图片或视频，便于人工复核。

## 2. 输入与输出定义

### 输入

- 图片输入：`jpg`、`jpeg`、`png` 等常见格式。
- 视频输入：`mp4`、`avi`、`mov` 等常见格式。
- 时间段参数：开始时间、结束时间或对应的视频帧范围。

### 输出

- 检测框：每头驴对应一个边界框，格式为 `[x_min, y_min, x_max, y_max, confidence, class]`。
- 单帧数量：当前图片或视频帧内检测到的驴数量。
- 时间段统计：
  - 每帧数量。
  - 时间段内平均数量。
  - 时间段内最大数量。
  - 时间段内最小数量。
  - 可选：按秒、分钟聚合后的数量变化曲线。
- 可视化文件：带检测框和计数结果的图片或视频。

## 3. 技术路线

### 3.1 基础方案

使用一阶段目标检测模型作为核心检测器，优先选择 YOLO 系列模型，原因如下：

- 推理速度快，适合监控场景。
- 训练和部署流程成熟。
- 对小目标和密集目标场景可以通过数据增强、切片推理和更高分辨率训练进行优化。
- 社区工具完善，便于标注、训练、评估和部署。

推荐初始模型：

- 开发验证：轻量模型，如 YOLO 的 nano/small 版本。
- 精度优先：中等模型，如 YOLO 的 medium 版本。
- 部署到边缘设备：根据设备算力选择轻量模型并导出 ONNX/TensorRT。

### 3.2 数据处理流程

1. 收集农场俯视视角监控图片和视频。
2. 从视频中按固定间隔抽帧，构建图片数据集。
3. 使用标注工具为每头驴绘制检测框。
4. 将数据集划分为训练集、验证集和测试集。
5. 训练目标检测模型。
6. 在测试集和真实监控片段上评估模型效果。
7. 优化误检、漏检、小目标识别和遮挡场景。
8. 封装图片、视频和时间段统计推理接口。

## 4. 数据集建设计划

### 4.1 数据来源

- 农场固定监控摄像头截图。
- 农场监控视频抽帧。
- 不同天气、光照、季节、地面背景和驴群密度下的样本。

### 4.2 数据规模建议

第一阶段可从小规模数据开始：

- 原始图片：500 到 1000 张。
- 标注目标数：尽量覆盖 3000 个以上驴实例。
- 场景覆盖：
  - 白天强光。
  - 阴天。
  - 黄昏或弱光。
  - 驴单独出现。
  - 驴密集聚集。
  - 驴互相遮挡。
  - 驴靠近围栏、棚舍、草料槽等容易混淆区域。

第二阶段根据错误样例继续补充数据，重点补充漏检和误检场景。

### 4.3 标注规范

- 类别名称统一为 `donkey`。
- 检测框尽量贴合驴的可见身体范围。
- 被遮挡的驴只标注可见部分，但需要保证框内主体明确。
- 过小、严重模糊、无法人工确认的目标不标注或单独记录。
- 标注完成后抽样复查，避免漏标、框偏移和类别错误。

### 4.4 数据划分

建议按场景划分，而不是随机按图片划分，避免同一段视频相邻帧同时出现在训练集和测试集：

- 训练集：70%
- 验证集：20%
- 测试集：10%

测试集应包含完整未参与训练的视频片段，用于评估真实泛化能力。

## 5. 模型训练计划

### 5.1 训练配置

建议从以下配置开始：

- 输入尺寸：`960` 或 `1280`，俯视监控中的驴可能较小，输入尺寸不宜过低。
- 批大小：根据显存调整。
- 训练轮数：初始 100 到 200 epoch。
- 预训练权重：使用通用目标检测预训练权重初始化。
- 数据增强：
  - 随机缩放。
  - 随机裁剪。
  - 亮度和对比度扰动。
  - 轻微旋转。
  - Mosaic 或 MixUp，视模型框架支持情况启用。

### 5.2 小目标优化

农场俯视图片中驴可能占画面比例较小，需重点关注小目标检测：

- 使用更高输入分辨率训练和推理。
- 对高分辨率监控图进行切片推理，再合并检测结果。
- 增加小目标样本和密集样本。
- 调整置信度阈值和 NMS 阈值，减少漏检。
- 对固定摄像头场景，可以裁剪掉天空、墙体等无关区域。

### 5.3 遮挡与密集场景优化

- 补充驴聚集、互相遮挡、围栏遮挡等样本。
- 标注时保持一致的遮挡处理规范。
- 使用跟踪算法辅助视频计数，减少单帧检测抖动。
- 对连续视频采用时间平滑，降低数量忽高忽低的问题。

## 6. 视频与时间段计数方案

### 6.1 单帧计数

对每一帧执行目标检测：

1. 读取图片或视频帧。
2. 模型输出所有驴检测框。
3. 过滤低置信度检测框。
4. 经过 NMS 去重。
5. 剩余检测框数量即当前画面驴数量。

### 6.2 时间段计数

对指定时间段的视频帧进行抽样或逐帧检测：

- 精确统计：逐帧检测，计算每帧数量。
- 快速统计：按固定 FPS 抽帧，例如每秒 1 到 5 帧。
- 输出统计值：
  - `frame_count`: 参与统计的帧数。
  - `avg_count`: 平均数量。
  - `max_count`: 最大数量。
  - `min_count`: 最小数量。
  - `count_by_time`: 每个时间点的数量。

### 6.3 可选跟踪方案

如果需要统计一段时间内出现过多少头不同的驴，而不是每个画面内有多少头驴，则需要引入多目标跟踪：

- 检测器：YOLO 系列模型。
- 跟踪器：ByteTrack、BoT-SORT 或 DeepSORT。
- 输出：每头驴的轨迹 ID、出现时间、离开时间。

当前 README 中描述的是“画面内的驴的数量”，因此第一阶段以单帧画面内数量统计为主。

## 7. 评估指标

### 7.1 检测指标

- Precision：检测为驴的目标中有多少是真的驴。
- Recall：真实驴中有多少被检测出来。
- mAP@0.5：IoU 阈值为 0.5 时的平均精度。
- mAP@0.5:0.95：多个 IoU 阈值下的平均精度。

### 7.2 计数指标

- MAE：平均绝对计数误差。
- RMSE：均方根计数误差。
- Accuracy within tolerance：误差在 1 头或 2 头以内的帧比例。

### 7.3 业务验收建议

第一阶段目标：

- 单帧计数 MAE 小于 1.0。
- 常规光照场景 Recall 大于 90%。
- 漏检和误检样例可追踪、可复盘。

第二阶段目标：

- 弱光、遮挡、密集场景稳定性提升。
- 视频数量曲线无明显抖动。
- 满足实际监控部署速度要求。

## 9. 推理接口设计

### 9.1 图片检测接口

输入：

```text
python scripts/infer_image.py --source image.jpg --weights models/best.pt --conf 0.25
```

输出：

- 带检测框图片。
- JSON 结果文件。

JSON 示例：

```json
{
  "source": "image.jpg",
  "count": 5,
  "detections": [
    {
      "class": "donkey",
      "confidence": 0.93,
      "bbox": [120, 80, 260, 210]
    }
  ]
}
```

### 9.2 视频时间段统计接口

输入：

```text
python scripts/infer_video.py \
  --source farm.mp4 \
  --weights models/best.pt \
  --start 00:01:00 \
  --end 00:03:00 \
  --sample-fps 2
```

输出：

- 带检测框和数量的视频。
- 时间段统计 JSON。
- 数量随时间变化的 CSV。

## 10. 开发里程碑


### 阶段一：需求确认与数据准备

如果已有真实监控视频，主要时间花在抽帧、筛选和标注；如果数据不足，需要额外增加采集周期。

- 明确输入类型：图片、视频或实时流。
- 明确输出形式：图片、JSON、CSV、视频或 API。
- 收集首批监控数据。
- 完成标注规范和数据集划分。

交付物：

- 标注数据集。
- 数据集说明文档。
- 初始 `dataset.yaml`。

### 阶段二：基线模型训练
首次训练和验证通常较快，重点是得到一版可评估的基线结果。

- 搭建训练环境。
- 训练第一版 YOLO 检测模型。
- 在验证集和测试集上评估。
- 输出错误样例。

交付物：

- 第一版模型权重。
- 评估报告。
- 可视化检测结果。

### 阶段三：计数与视频推理
包括图片推理、视频时间段统计、结果导出和可视化。

- 实现图片检测脚本。
- 实现视频时间段统计脚本。
- 实现结果 JSON、CSV 和可视化输出。
- 评估计数误差。

交付物：

- 推理脚本。
- 时间段计数报告。
- 示例输出文件。

### 阶段四：优化与部署
时间主要取决于漏检、误检、遮挡、小目标和部署设备性能要求。

- 根据漏检和误检样例补充数据。
- 优化训练参数和推理阈值。
- 可选引入切片推理和多目标跟踪。
- 根据部署设备导出模型格式。

交付物：

- 优化后的模型。
- 部署说明。
- 推理性能报告。


## 12. 风险与应对

| 风险 | 影响 | 应对方案 |
| --- | --- | --- |
| 数据量不足 | 模型泛化差 | 持续补充不同场景数据 |
| 俯视角下驴目标较小 | 漏检增加 | 高分辨率训练、切片推理、小目标增强 |
| 驴互相遮挡 | 计数偏低 | 增加遮挡样本，引入视频跟踪和平滑 |
| 光照变化大 | 误检或漏检 | 增加弱光/强光数据和颜色增强 |
| 背景中有相似物体 | 误检增加 | 补充负样本，复查错误样例 |
| 视频帧间抖动 | 数量曲线不稳定 | 时间平滑或多目标跟踪 |

## 13. 下一步建议

优先完成以下工作：

1. 收集 5 到 10 段真实农场俯视监控视频。
2. 从视频中抽取 500 张左右有代表性的图片。
3. 使用统一规范标注驴检测框。
4. 建立数据集目录和 `dataset.yaml`。
5. 训练第一版基线模型。
6. 基于错误样例迭代数据和模型。
